## Overview

The Dart Runtime provides the HTTP client for making API calls from Dart and Flutter applications. It supports iOS, Android, web, and desktop platforms with automatic device information collection and type-safe request/response handling.

## Installation

Add to your `pubspec.yaml`:

```yaml
dependencies:
  sdkgen_runtime: ^latest
```

## SdkgenHttpClient

The main class for making HTTP requests from Dart/Flutter applications.

### Constructor

```dart
SdkgenHttpClient(
  String baseUrl,
  http.Client? client,
  Map<String, Object> typeTable,
  Map<String, FunctionDescription> fnTable,
  Map<String, SdkgenErrorDescription> errTable,
)
```

<ParamField path="baseUrl" type="String" required>
  The base URL of your sdkgen API server (e.g., `https://api.example.com`)
</ParamField>

<ParamField path="client" type="http.Client?">
  Optional HTTP client instance. If null, a default client is created. Pass a custom client for testing or to configure proxy/timeout settings.
</ParamField>

<ParamField path="typeTable" type="Map<String, Object>" required>
  Type definitions table generated by sdkgen
</ParamField>

<ParamField path="fnTable" type="Map<String, FunctionDescription>" required>
  Function descriptions table mapping function names to their argument and return types
</ParamField>

<ParamField path="errTable" type="Map<String, SdkgenErrorDescription>" required>
  Error descriptions table for creating typed error instances
</ParamField>

### Properties

#### extra

```dart
final Map<String, dynamic> extra
```

A map for storing extra metadata to be sent with every request. Commonly used for authentication tokens and session data.

**Example:**
```dart
client.extra['authToken'] = 'abc123';
client.extra['userId'] = 12345;
```

#### headers

```dart
final Map<String, String> headers
```

HTTP headers sent with each request. Defaults to `{'Content-Type': 'application/sdkgen'}`. You can add custom headers:

**Example:**
```dart
client.headers['X-Custom-Header'] = 'value';
client.headers['Authorization'] = 'Bearer token';
```

#### interceptors

```dart
final List<SdkgenInterceptors> interceptors
```

List of interceptors for request/response/error handling. See [Interceptors](#interceptors-1) section for details.

#### deviceId

```dart
String? deviceId
```

The device identifier. Automatically generated and persisted using SharedPreferences on first use.

### Methods

#### makeRequest

```dart
Future<Object?> makeRequest(
  String functionName,
  Map<String, Object?> args,
) async
```

Makes an HTTP request to call a remote function on your sdkgen API.

<ParamField path="functionName" type="String" required>
  The name of the API function to call
</ParamField>

<ParamField path="args" type="Map<String, Object?>" required>
  A map of argument names to values
</ParamField>

<ResponseField name="return" type="Future<Object?>">
  A Future that resolves with the decoded API response
</ResponseField>

**Behavior:**
- Encodes arguments according to the function's type signature
- Automatically includes device information
- Generates and persists a unique device ID in SharedPreferences
- Sends extra metadata from the `extra` map
- Calls interceptors at appropriate lifecycle points
- Decodes and validates the response
- Throws typed `SdkgenError` on failure

**Example:**
```dart
try {
  final result = await client.makeRequest('getUser', {'id': 123});
  print('User data: $result');
} on SdkgenErrorWithData<ValidationData> catch (e) {
  print('Validation error: ${e.message}');
  print('Invalid fields: ${e.data.fields}');
} on SdkgenError catch (e) {
  print('API error: ${e.message}');
}
```

#### addInterceptor

```dart
SdkgenHttpClient addInterceptor(SdkgenInterceptors interceptor)
```

Adds an interceptor to the client. Returns the client instance for method chaining.

<ParamField path="interceptor" type="SdkgenInterceptors" required>
  The interceptor instance to add
</ParamField>

**Example:**
```dart
client
  .addInterceptor(LoggingInterceptor())
  .addInterceptor(AuthInterceptor());
```

## Interceptors

Interceptors allow you to hook into the request/response lifecycle.

### SdkgenInterceptors

Abstract class for creating custom interceptors.

```dart
abstract class SdkgenInterceptors {
  OnRequestInterceptor? get onRequest;
  OnResponseInterceptor? get onResponse;
  OnErrorInterceptor? get onError;
}
```

### Interceptor Types

#### OnRequestInterceptor

```dart
typedef OnRequestInterceptor = Future Function(Map<String, dynamic> body);
```

Called before each request is sent. Receives the full request body including encoded args, device info, and extra fields.

#### OnResponseInterceptor

```dart
typedef OnResponseInterceptor = Future Function(dynamic response);
```

Called after successful responses are received and decoded.

#### OnErrorInterceptor

```dart
typedef OnErrorInterceptor = Future Function(dynamic error);
```

Called when requests fail or errors are returned from the API.

### Example Interceptor

```dart
class LoggingInterceptor implements SdkgenInterceptors {
  @override
  OnRequestInterceptor? get onRequest => (body) async {
    print('Request: ${body['name']} with args ${body['args']}');
  };

  @override
  OnResponseInterceptor? get onResponse => (response) async {
    print('Response received: $response');
  };

  @override
  OnErrorInterceptor? get onError => (error) async {
    print('Error occurred: $error');
  };
}
```

## Error Classes

### SdkgenError

Base exception class for all sdkgen errors.

```dart
class SdkgenError implements Exception {
  final String message;
  final Json request;
  const SdkgenError(this.message, [this.request = const {}]);
}
```

<ParamField path="message" type="String">
  The error message
</ParamField>

<ParamField path="request" type="Json">
  Request information including requestId and function name
</ParamField>

### SdkgenErrorWithData

Exception class for errors that include additional typed data.

```dart
class SdkgenErrorWithData<T> extends SdkgenError {
  final T data;
  const SdkgenErrorWithData(String message, Json request, this.data);
}
```

**Example:**
```dart
try {
  await client.makeRequest('createUser', {'email': 'invalid'});
} on SdkgenErrorWithData<ValidationErrorData> catch (e) {
  print('Validation failed: ${e.message}');
  print('Invalid fields: ${e.data.fields}');
  print('Request ID: ${e.request['requestId']}');
}
```

## Device Information

The Dart runtime automatically collects platform-specific device information:

### iOS/Android

```dart
{
  'id': String,        // Persistent device ID from SharedPreferences
  'platform': {
    'os': String,      // 'ios' or 'android'
    'version': String, // OS version
    'model': String,   // Device model
  },
  'language': String,  // Device language code
  'type': String,      // 'mobile'
}
```

### Web

```dart
{
  'id': String,              // Persistent device ID
  'platform': {
    'userAgent': String,     // Browser user agent
  },
  'language': String,        // Browser language
  'type': String,            // 'web'
}
```

## Type Encoding/Decoding

The runtime provides automatic encoding and decoding for sdkgen types:

### Supported Types

- **Primitives**: `string`, `bool`, `int`, `uint`, `float`, `money`
- **Special strings**: `email`, `cpf`, `cnpj`, `url`, `html`, `xml`, `hex`, `uuid`, `base64`
- **Complex types**: `bytes`, `bigint`, `decimal`, `date`, `datetime`, `json`
- **Collections**: Arrays (`type[]`), optional types (`type?`)
- **Custom types**: Structs and enums defined in your API schema

### Type Mapping

| sdkgen Type | Dart Type |
|-------------|----------|
| `string` | `String` |
| `bool` | `bool` |
| `int` | `int` |
| `float` | `double` |
| `bytes` | `List<int>` |
| `bigint` | `BigInt` |
| `decimal` | `Decimal` |
| `date` | `DateTime` |
| `datetime` | `DateTime` |
| `json` | `dynamic` |

## Platform-Specific Considerations

### Flutter

**Permissions**: No special permissions required for network access on most platforms.

**iOS**: Add to `ios/Runner/Info.plist` if accessing HTTP endpoints:
```xml
<key>NSAppTransportSecurity</key>
<dict>
  <key>NSAllowsArbitraryLoads</key>
  <true/>
</dict>
```

**Android**: Add to `android/app/src/main/AndroidManifest.xml`:
```xml
<uses-permission android:name="android.permission.INTERNET" />
```

### Testing

Pass a mock HTTP client for testing:

```dart
import 'package:http/testing.dart';

final mockClient = MockClient((request) async {
  return Response('{"result": "mocked"}', 200);
});

final client = SdkgenHttpClient(
  'https://api.example.com',
  mockClient,
  typeTable,
  fnTable,
  errTable,
);
```

## Complete Example

```dart
import 'package:sdkgen_runtime/sdkgen_runtime.dart';
import 'api_types.dart'; // Generated by sdkgen

// Define custom error classes
class ValidationError extends SdkgenErrorWithData<ValidationErrorData> {
  const ValidationError(String msg, Json req, ValidationErrorData data)
      : super(msg, req, data);
}

class AuthenticationError extends SdkgenError {
  const AuthenticationError(String msg, Json req) : super(msg, req);
}

// Create error table
final errTable = {
  'ValidationError': SdkgenErrorDescription(
    'ValidationErrorData',
    (msg, req, data) => ValidationError(msg, req, data),
  ),
  'AuthenticationError': SdkgenErrorDescription(
    'void',
    (msg, req, data) => AuthenticationError(msg, req),
  ),
  'Fatal': SdkgenErrorDescription(
    'void',
    (msg, req, data) => SdkgenError(msg, req),
  ),
};

// Initialize client
final api = SdkgenHttpClient(
  'https://api.example.com',
  null, // Use default HTTP client
  typeTable,
  fnTable,
  errTable,
);

void main() async {
  // Set authentication token
  api.extra['authToken'] = 'abc123';

  // Add logging interceptor
  api.addInterceptor(LoggingInterceptor());

  // Make API calls
  try {
    final user = await api.makeRequest('getUser', {'id': 123});
    print('User: $user');

    final users = await api.makeRequest('listUsers', {
      'page': 1,
      'limit': 10,
    });
    print('Users: $users');
  } on ValidationError catch (e) {
    print('Validation error: ${e.message}');
    print('Invalid fields: ${e.data.fields}');
  } on AuthenticationError catch (e) {
    print('Authentication failed: ${e.message}');
    // Navigate to login screen
  } on SdkgenError catch (e) {
    print('API error: ${e.message}');
  }
}
```
