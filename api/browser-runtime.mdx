## Overview

The Browser Runtime provides the HTTP client for making API calls from web browsers using TypeScript/JavaScript. It handles request encoding, device information collection, error handling, and response decoding.

## Installation

```bash
npm install @sdkgen/browser-runtime
```

## SdkgenHttpClient

The main class for making HTTP requests from browser environments.

### Constructor

```typescript
new SdkgenHttpClient(
  baseUrl: string,
  astJson: DeepReadonly<AstJson>,
  errClasses: ErrClasses
)
```

<ParamField path="baseUrl" type="string" required>
  The base URL of your sdkgen API server (e.g., `https://api.example.com`)
</ParamField>

<ParamField path="astJson" type="DeepReadonly<AstJson>" required>
  The Abstract Syntax Tree JSON generated by sdkgen, containing type definitions and function signatures
</ParamField>

<ParamField path="errClasses" type="ErrClasses" required>
  A record mapping error type names to error class constructors for proper error handling
</ParamField>

### Properties

#### extra

```typescript
extra: Map<string, unknown>
```

A Map for storing extra metadata to be sent with every request. Useful for authentication tokens, session data, or custom headers.

**Example:**
```typescript
client.extra.set("authToken", "abc123");
client.extra.set("sessionId", "xyz789");
```

#### successHook

```typescript
successHook: (result: any, name: string, args: any) => void
```

A callback function invoked after successful API calls. Use this for logging, analytics, or post-processing.

<ParamField path="result" type="any">
  The decoded response result from the API
</ParamField>

<ParamField path="name" type="string">
  The name of the function that was called
</ParamField>

<ParamField path="args" type="any">
  The arguments passed to the function
</ParamField>

**Example:**
```typescript
client.successHook = (result, name, args) => {
  console.log(`Successfully called ${name}`, { args, result });
};
```

#### errorHook

```typescript
errorHook: (error: any, name: string, args: any) => void
```

A callback function invoked when API calls fail. Use this for error logging, monitoring, or custom error handling.

<ParamField path="error" type="any">
  The error object returned from the API
</ParamField>

<ParamField path="name" type="string">
  The name of the function that was called
</ParamField>

<ParamField path="args" type="any">
  The arguments passed to the function
</ParamField>

**Example:**
```typescript
client.errorHook = (error, name, args) => {
  console.error(`Error calling ${name}:`, error);
  // Send to error tracking service
};
```

### Methods

#### makeRequest

```typescript
async makeRequest(functionName: string, args: unknown): Promise<any>
```

Makes an HTTP request to call a remote function on your sdkgen API.

<ParamField path="functionName" type="string" required>
  The name of the API function to call
</ParamField>

<ParamField path="args" type="unknown" required>
  The arguments to pass to the function
</ParamField>

<ResponseField name="return" type="Promise<any>">
  A promise that resolves with the decoded API response
</ResponseField>

**Behavior:**
- Encodes arguments according to the function's type signature
- Automatically includes device information (browser user agent, language, timezone)
- Generates a unique device ID stored in localStorage
- Sends extra metadata from the `extra` Map
- Uses XMLHttpRequest for maximum browser compatibility
- Decodes the response and validates against expected types
- Throws typed errors on failure

**Example:**
```typescript
try {
  const result = await client.makeRequest("getUser", { id: 123 });
  console.log("User data:", result);
} catch (error) {
  if (error instanceof CustomApiError) {
    console.error("API error:", error.message);
  }
}
```

## Device Information

The browser runtime automatically collects and sends device information with each request:

```typescript
{
  id: string,              // Persistent device ID from localStorage
  language: string,        // Browser language (navigator.language)
  platform: {
    browserUserAgent: string  // Full user agent string
  },
  timezone: string | null, // IANA timezone (e.g., "America/New_York")
  type: "web",
  version: string          // Script source URL
}
```

## Error Classes

### SdkgenError

Base error class for all sdkgen errors.

```typescript
abstract class SdkgenError extends Error {
  get type(): string;
  toJSON(): { message: string; type: string };
}
```

### SdkgenErrorWithData

Error class for errors that include additional data.

```typescript
abstract class SdkgenErrorWithData<DataType> extends SdkgenError {
  data: DataType;
  toJSON(): { data: DataType; message: string; type: string };
}
```

**Example:**
```typescript
class ValidationError extends SdkgenErrorWithData<{ field: string }> {}
class NotFoundError extends SdkgenError {}

const errClasses = {
  ValidationError,
  NotFoundError,
  Fatal: SdkgenError
};
```

## Platform-Specific Considerations

### Browser Compatibility

- Uses XMLHttpRequest for broad browser support (IE10+)
- Gracefully handles localStorage unavailability (private browsing)
- Falls back to in-memory device ID if localStorage is blocked

### CORS Requirements

Your sdkgen server must be configured to accept cross-origin requests:

```javascript
// Server-side CORS headers required
Access-Control-Allow-Origin: *
Access-Control-Allow-Headers: Content-Type
Access-Control-Allow-Methods: POST
```

### Content Security Policy

If using CSP, ensure you allow connections to your API domain:

```html
<meta http-equiv="Content-Security-Policy" 
      content="connect-src 'self' https://api.example.com">
```

## Complete Example

```typescript
import { SdkgenHttpClient, SdkgenError, SdkgenErrorWithData } from '@sdkgen/browser-runtime';
import astJson from './api.json';

// Define custom error classes
class ValidationError extends SdkgenErrorWithData<{ fields: string[] }> {}
class AuthenticationError extends SdkgenError {}
class NotFoundError extends SdkgenError {}

// Create error classes mapping
const errClasses = {
  ValidationError,
  AuthenticationError,
  NotFoundError,
  Fatal: SdkgenError
};

// Initialize the client
const api = new SdkgenHttpClient(
  'https://api.example.com',
  astJson,
  errClasses
);

// Set authentication token
api.extra.set('authToken', localStorage.getItem('token'));

// Add logging hooks
api.successHook = (result, name) => {
  console.log(`✓ ${name} succeeded`);
};

api.errorHook = (error, name) => {
  console.error(`✗ ${name} failed:`, error);
};

// Make API calls
async function example() {
  try {
    const user = await api.makeRequest('getUser', { id: 123 });
    console.log('User:', user);
  } catch (error) {
    if (error instanceof ValidationError) {
      console.error('Invalid fields:', error.data.fields);
    } else if (error instanceof AuthenticationError) {
      // Redirect to login
      window.location.href = '/login';
    } else {
      console.error('Unexpected error:', error);
    }
  }
}
```

## TypeScript Support

The browser runtime is written in TypeScript and includes full type definitions. When you generate your API client with sdkgen, the types are automatically inferred from your API schema.

```typescript
// Generated client has typed methods
const result = await api.getUser({ id: 123 });
//    ^? result is inferred as User type
```
