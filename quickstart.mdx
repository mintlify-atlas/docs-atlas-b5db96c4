---
title: Quickstart
description: Get started with sdkgen in minutes and build your first type-safe API
---

Build your first API with sdkgen in just a few minutes. This guide will walk you through installing sdkgen, defining your API, generating code, and implementing a working server.

<Steps>
  <Step title="Install the CLI">
    Install the sdkgen CLI globally using your preferred package manager:

    <CodeGroup>
    ```bash npm
    npm install -g @sdkgen/cli
    ```

    ```bash yarn
    yarn global add @sdkgen/cli
    ```

    ```bash pnpm
    pnpm add -g @sdkgen/cli
    ```
    </CodeGroup>

    <Check>Verify installation by running `sdkgen --help`</Check>
  </Step>

  <Step title="Create your API definition">
    Create a file named `api.sdkgen` to describe your API:

    ```sdkgen api.sdkgen
    type Post {
      id: uuid
      title: string
      body: string
      createdAt: datetime
      author: {
        name: string
        email: email
      }
    }

    fn getPost(id: uuid): Post?
    fn createPost(title: string, body: string, authorName: string, authorEmail: email): Post
    fn listPosts(): Post[]
    ```

    This defines a simple blog API with three functions: getting a post by ID, creating a new post, and listing all posts.
  </Step>

  <Step title="Generate server code">
    Generate TypeScript server code from your API definition:

    ```bash
    sdkgen api.sdkgen -o api.ts -t typescript_nodeserver
    ```

    This creates `api.ts` with type-safe server code including:
    - TypeScript interfaces for all types
    - Type validation for all requests and responses
    - Function handlers that you need to implement
  </Step>

  <Step title="Set up your project">
    Initialize a Node.js project and install dependencies:

    ```bash
    npm init -y
    npm install @sdkgen/node-runtime
    npm install --save-dev typescript @types/node
    npx tsc --init --target esnext
    ```
  </Step>

  <Step title="Implement your server">
    Create `server.ts` and implement the API functions:

    ```typescript server.ts
    import { SdkgenHttpServer } from "@sdkgen/node-runtime";
    import { api } from "./api";
    import { randomUUID } from "crypto";

    // In-memory storage
    const posts = new Map();

    // Implement getPost
    api.fn.getPost = async (ctx, { id }) => {
      const post = posts.get(id);
      return post || null; // Return null if not found
    };

    // Implement createPost
    api.fn.createPost = async (ctx, { title, body, authorName, authorEmail }) => {
      const post = {
        id: randomUUID(),
        title,
        body,
        createdAt: new Date(),
        author: {
          name: authorName,
          email: authorEmail
        }
      };
      
      posts.set(post.id, post);
      return post;
    };

    // Implement listPosts
    api.fn.listPosts = async (ctx, {}) => {
      return Array.from(posts.values());
    };

    // Start the server
    const server = new SdkgenHttpServer(api, {});
    server.listen(8000);
    console.log("Server running on http://localhost:8000");
    ```

    <Note>
      All function arguments are automatically validated against your API definition. Invalid requests are rejected before reaching your code.
    </Note>
  </Step>

  <Step title="Run your server">
    Compile and run your server:

    ```bash
    npx tsc
    node server.js
    ```

    Your API is now running on `http://localhost:8000`!
  </Step>

  <Step title="Test in the playground">
    sdkgen includes a built-in interactive playground. Open your browser to:

    ```
    http://localhost:8000
    ```

    You'll see an interface where you can:
    - Browse all available functions
    - Test function calls with real parameters
    - See request/response data
    - View generated code for different platforms

    Try creating a post:
    ```json
    {
      "title": "My First Post",
      "body": "This is a test post created via sdkgen!",
      "authorName": "John Doe",
      "authorEmail": "john@example.com"
    }
    ```
  </Step>
</Steps>

## Advanced example with error handling

Enhance your API with custom errors and validation:

```sdkgen api.sdkgen
error NotFound
error InvalidInput {
  field: string
  reason: string
}

type Post {
  id: uuid
  title: string
  body: string
  createdAt: datetime
  author: {
    name: string
    email: email
  }
}

@description("Get a post by its ID")
@throws NotFound
fn getPost(id: uuid): Post

@description("Create a new post")
@throws InvalidInput
fn createPost(title: string, body: string, authorName: string, authorEmail: email): Post

@rest GET /posts
fn listPosts(): Post[]
```

Implement with error handling:

```typescript server.ts
import { SdkgenError } from "@sdkgen/node-runtime";

api.fn.getPost = async (ctx, { id }) => {
  const post = posts.get(id);
  if (!post) {
    throw new SdkgenError("NotFound", `Post with id ${id} not found`);
  }
  return post;
};

api.fn.createPost = async (ctx, { title, body, authorName, authorEmail }) => {
  if (title.length < 3) {
    throw new SdkgenError("InvalidInput", "Title too short", {
      field: "title",
      reason: "Must be at least 3 characters"
    });
  }

  const post = {
    id: randomUUID(),
    title,
    body,
    createdAt: new Date(),
    author: { name: authorName, email: authorEmail }
  };
  
  posts.set(post.id, post);
  return post;
};
```

<Info>
  The `@rest` annotation automatically creates a REST endpoint at `GET /posts`. The playground will show both the RPC and REST interfaces.
</Info>
